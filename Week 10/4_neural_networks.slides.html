The training loop now becomes¶

In [37]:
def training_loop(n_epochs, optimizer, model, loss_fn, t_u_train, t_u_val, t_c_train, t_c_val):
    for epoch in range(1, n_epochs + 1):
        t_p_train = model(t_u_train) # <1>
        loss_train = loss_fn(t_p_train, t_c_train)

        t_p_val = model(t_u_val) # <1>
        loss_val = loss_fn(t_p_val, t_c_val)
        
        optimizer.zero_grad()
        loss_train.backward() # <2>
        optimizer.step()

        if epoch == 1 or epoch % 1000 == 0:
            print('Epoch {}, Training loss {}, Validation loss {}'.format(
                epoch, float(loss_train), float(loss_val)))

Note how parameters are no longer passed to the loop, model or loss but contained in the model¶

Here we have a linear function whos output is passed into tanh activation function and then again into a linear function.¶

seq_model = nn.Sequential(
            nn.Linear(1, 13), # <1>
            nn.Tanh(),
            nn.Linear(13, 1)) # <2>
seq_model

  (2): Linear(in_features=13, out_features=1, bias=True)

Note the arbitrarily chosen number 13 at <1> and <2>. The idea is that we make the number of features large internally in the model and then compress them back down to a single output. In that way we can learn different aspects of the data that are relevant for the overall output.

What if the model cannot be written as a sequential process, using nn.Sequential?¶

Answer: subclass the nn.Module¶

In [48]:
class SubclassModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.hidden_linear = nn.Linear(1, 13)
        self.hidden_activation = nn.Tanh()
        self.output_linear = nn.Linear(13, 1)
        
    def forward(self, input):
        hidden_t = self.hidden_linear(input)
        activated_t = self.hidden_activation(hidden_t)
        output_t = self.output_linear(activated_t)
        return output_t

subclass_model = SubclassModel()
subclass_model
